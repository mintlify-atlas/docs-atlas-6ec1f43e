title: "Loading and managing images in Ebitengine"
description: "How to load PNG and other images from disk, embed them with go:embed, use sprite sheets with SubImage, and draw to offscreen render textures."
---

Images are the primary visual primitive in Ebitengine. Every texture — sprite, background, UI element, or offscreen render target — is an `*ebiten.Image`. This guide covers the full lifecycle: loading images from disk or embedded data, slicing sprite sheets into sub-images, creating images programmatically, and using offscreen images as render textures.

## Loading images from a file

The `ebitenutil` package provides `NewImageFromFile`, which opens a file by path and returns an `*ebiten.Image`. It works with PNG, JPEG, and any format registered with the standard `image` package.

```go
import "github.com/hajimehoshi/ebiten/v2/ebitenutil"

image, _, err := ebitenutil.NewImageFromFile("assets/player.png")
if err != nil {
    log.Fatal(err)
}
```

<Warning>
  `NewImageFromFile` reads from the filesystem at the path you provide. In a distributed game binary, use `go:embed` (see below) so your assets are bundled inside the executable.
</Warning>

## Embedding images with go:embed

Go 1.16+ lets you embed files directly into the binary using the `//go:embed` directive. This is the recommended approach for shipping game assets.

```go
import (
    "bytes"
    _ "embed"
    "image/png"

    "github.com/hajimehoshi/ebiten/v2"
)

//go:embed assets/player.png
var playerPNG []byte

func loadPlayer() (*ebiten.Image, error) {
    img, err := png.Decode(bytes.NewReader(playerPNG))
    if err != nil {
        return nil, err
    }
    return ebiten.NewImageFromImage(img), nil
}
```

For larger asset trees, embed an entire directory and use `embed.FS`:

```go
import (
    "embed"
    "image/png"

    "github.com/hajimehoshi/ebiten/v2"
    "github.com/hajimehoshi/ebiten/v2/ebitenutil"
)

//go:embed assets
var assets embed.FS

func loadFromFS(name string) (*ebiten.Image, error) {
    f, err := assets.Open(name)
    if err != nil {
        return nil, err
    }
    defer f.Close()
    return ebitenutil.NewImageFromReader(f)
}
```

<Tip>
  Embedding a directory with `//go:embed assets` includes all files recursively. Use `embed.FS` with `fs.ReadDir` to list available assets at runtime.
</Tip>

## Loading from an io.Reader

`ebitenutil.NewImageFromReader` accepts any `io.Reader`, making it easy to load images from in-memory buffers, HTTP responses, or zip archives.

```go
import (
    "bytes"
    "github.com/hajimehoshi/ebiten/v2/ebitenutil"
)

func loadFromBytes(data []byte) (*ebiten.Image, error) {
    return ebitenutil.NewImageFromReader(bytes.NewReader(data))
}
```

Alternatively, decode using the standard `image` package and convert:

```go
import (
    "image/png"
    "os"

    "github.com/hajimehoshi/ebiten/v2"
)

func loadPNG(path string) (*ebiten.Image, error) {
    f, err := os.Open(path)
    if err != nil {
        return nil, err
    }
    defer f.Close()

    src, err := png.Decode(f)
    if err != nil {
        return nil, err
    }
    return ebiten.NewImageFromImage(src), nil
}
```

## Sprite sheets and sub-images

Packing multiple sprites into a single image (a sprite sheet) reduces GPU state changes. Use `SubImage` to reference a rectangular region of a larger image without copying pixel data.

```go
import (
    "image"
    "github.com/hajimehoshi/ebiten/v2"
    "github.com/hajimehoshi/ebiten/v2/ebitenutil"
)

const frameSize = 32

type Sprite struct {
    sheet  *ebiten.Image
    frames []image.Rectangle
}

func NewSprite(path string, cols, rows int) (*Sprite, error) {
    sheet, _, err := ebitenutil.NewImageFromFile(path)
    if err != nil {
        return nil, err
    }

    var frames []image.Rectangle
    for row := 0; row < rows; row++ {
        for col := 0; col < cols; col++ {
            r := image.Rect(
                col*frameSize, row*frameSize,
                (col+1)*frameSize, (row+1)*frameSize,
            )
            frames = append(frames, r)
        }
    }
    return &Sprite{sheet: sheet, frames: frames}, nil
}

func (s *Sprite) Frame(index int) *ebiten.Image {
    return s.sheet.SubImage(s.frames[index]).(*ebiten.Image)
}
```

Draw a specific animation frame:

```go
opts := &ebiten.DrawImageOptions{}
opts.GeoM.Translate(playerX, playerY)
screen.DrawImage(sprite.Frame(currentFrame), opts)
```

<Note>
  `SubImage` returns an `image.Image` interface. Assert it to `*ebiten.Image` with a type assertion before passing to `DrawImage`.
</Note>

## Creating images programmatically

`ebiten.NewImage(width, height int)` creates a blank GPU-backed image. Use it for offscreen render targets, procedurally generated textures, or any image whose contents you fill in code.

```go
// Solid-color rectangle
red := ebiten.NewImage(64, 64)
red.Fill(color.RGBA{R: 200, G: 40, B: 40, A: 255})

// Gradient (set individual pixels via image/draw)
import (
    "image"
    "image/color"
    "image/draw"
)

src := image.NewNRGBA(image.Rect(0, 0, 256, 256))
for y := 0; y < 256; y++ {
    for x := 0; x < 256; x++ {
        src.Set(x, y, color.NRGBA{R: uint8(x), G: uint8(y), B: 128, A: 255})
    }
}
gradient := ebiten.NewImageFromImage(src)
```

## Offscreen images (render textures)

You can draw to any `*ebiten.Image`, not just the `screen` parameter in `Draw`. Drawing to an offscreen image first, then blitting the result to the screen, is a powerful technique for post-processing, static caching, and multi-pass rendering.

```go
type Game struct {
    offscreen *ebiten.Image
    dirty     bool
}

func NewGame() *Game {
    return &Game{
        offscreen: ebiten.NewImage(screenWidth, screenHeight),
        dirty:     true,
    }
}

func (g *Game) Draw(screen *ebiten.Image) {
    if g.dirty {
        // Render expensive content once
        g.offscreen.Clear()
        drawExpensiveBackground(g.offscreen)
        g.dirty = false
    }

    // Blit the cached result every frame
    screen.DrawImage(g.offscreen, nil)

    // Draw dynamic content on top
    drawPlayers(screen)
}
```

<Tip>
  Use `image.Clear()` to reset an offscreen image to fully transparent before re-drawing to it. Without clearing, new draws are composited on top of previous content.
</Tip>

## Working with the standard image package

`ebiten.NewImageFromImage` accepts any `image.Image`, so you can use the full Go standard library for image manipulation before uploading to the GPU.

```go
import (
    "image"
    "image/color"
    "image/draw"

    "github.com/hajimehoshi/ebiten/v2"
)

// Create a standard library image
src := image.NewRGBA(image.Rect(0, 0, 100, 100))
draw.Draw(src, src.Bounds(), &image.Uniform{color.RGBA{0, 100, 200, 255}}, image.Point{}, draw.Src)

// Upload to GPU
ebitenImg := ebiten.NewImageFromImage(src)
```

<Note>
  `ebiten.NewImageFromImage` copies pixel data to GPU memory. After the call, you can discard the original `image.Image`. Changes to the source image after conversion are not reflected in the `ebiten.Image`.
</Note>
